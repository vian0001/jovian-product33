TOPOLOGI JARINGAN WIRELESS

Tampilan topologi di gns3

Topologi jaringan wireless menunjukkan bagaimana perangkat MikroTik terhubung dengan klien nirkabel. Konfigurasi security profile dan enkripsi WPA2 merupakan komponen penting untuk mengamankan jaringan nirkabel dari akses tidak sah.

Router MikroTik akan berfungsi sebagai Access Point (AP) yang menyediakan koneksi nirkabel terenkripsi bagi perangkat klien. Security profile menentukan parameter keamanan seperti metode autentikasi dan jenis enkripsi yang digunakan pada jaringan wireless.

Berikut adalah komponen-komponen utama dalam topologi ini:

1. MikroTik Router
# Berfungsi sebagai wireless access point dengan security profile WPA2.
2. Klien Wireless (Laptop, Smartphone, IoT devices)
# Perangkat yang terhubung ke jaringan nirkabel melalui autentikasi WPA2.
3. SSID (Service Set Identifier)
# Nama jaringan nirkabel yang terlihat oleh perangkat klien.

Konfigurasi security profile yang tepat sangat penting untuk mencegah serangan terhadap jaringan wireless. Menggunakan standar WPA2 dengan passphrase yang kuat adalah praktik keamanan dasar untuk melindungi jaringan Anda dari akses tidak sah.

1. SECURITY PROFILE DAN STANDAR WPA2

Pengertian Security Profile pada MikroTik

Dari sumber MikroTik Documentation (2024), Security Profile pada RouterOS adalah serangkaian konfigurasi keamanan yang diterapkan pada interface wireless untuk mengontrol bagaimana perangkat klien dapat terhubung dan mengakses jaringan. Security profile mencakup berbagai parameter seperti metode autentikasi, jenis enkripsi, dan kunci keamanan yang digunakan pada jaringan wireless.

Dari sumber NetworkLessons (2023), Security Profile pada MikroTik berfungsi sebagai templat keamanan yang dapat diterapkan ke satu atau beberapa interface wireless. Ini memungkinkan administrator untuk mengelola keamanan jaringan nirkabel secara terpusat dan konsisten, terutama pada perangkat MikroTik dengan multiple wireless interface.

Standar Enkripsi WPA2

Dari sumber TechTarget (2024), WPA2 (Wi-Fi Protected Access 2) adalah standar keamanan wireless yang dikembangkan oleh Wi-Fi Alliance sebagai penerus WPA. WPA2 menggunakan algoritma enkripsi AES (Advanced Encryption Standard) yang jauh lebih kuat dibandingkan dengan TKIP yang digunakan pada WPA generasi pertama. Standar WPA2 resmi dirilis pada tahun 2004 dan menjadi mandatory pada semua perangkat bersertifikasi Wi-Fi sejak 2006.

Dari sumber CBT Nuggets (2023), WPA2 mendukung dua mode utama: WPA2-Personal (WPA2-PSK) yang menggunakan pre-shared key dan cocok untuk jaringan rumah atau usaha kecil, serta WPA2-Enterprise yang menggunakan server RADIUS untuk autentikasi dan lebih cocok untuk lingkungan bisnis skala besar. WPA2 dengan AES-CCMP menawarkan keamanan yang sangat kuat dan telah menjadi standar industri selama bertahun-tahun sebelum kedatangan WPA3.

Dari 2 pendapat tersebut dapat disimpulkan bahwa Security Profile pada MikroTik adalah komponen konfigurasi keamanan yang penting untuk mengamankan jaringan wireless, sedangkan WPA2 adalah standar keamanan yang menggunakan enkripsi AES yang kuat untuk melindungi data pada jaringan nirkabel, tersedia dalam mode Personal (PSK) untuk implementasi sederhana dan Enterprise untuk lingkungan bisnis yang memerlukan keamanan tingkat tinggi.

2. MENGKONFIGURASI SECURITY PROFILE WPA2

Tampilan topologi di gns3

Membuat Security Profile WPA2 melalui CLI

Dari sumber Garrett's Mindless Rambling (2022), untuk membuat security profile WPA2 di MikroTik melalui Command Line Interface (CLI), Anda dapat menggunakan perintah yang sederhana namun efektif. Berikut adalah langkah-langkah untuk membuat security profile WPA2-PSK melalui CLI:

/interface wireless security-profiles
add name="profile-wpa2" authentication-types=wpa2-psk mode=dynamic-keys wpa2-pre-shared-key="password123"
# Membuat security profile bernama "profile-wpa2" dengan autentikasi WPA2-PSK

Dari sumber ShellHacks (2022), untuk konfigurasi yang lebih lengkap dengan dukungan kompatibilitas yang lebih baik, Anda dapat menambahkan parameter tambahan. Pendekatan yang direkomendasikan adalah sebagai berikut:

/interface wireless security-profiles
add name="secure-wifi" authentication-types=wpa2-psk mode=dynamic-keys
unicast-ciphers=aes-ccm group-ciphers=aes-ccm wpa2-pre-shared-key="StrongPassw0rd!"
# Membuat security profile dengan pengaturan cipher yang eksplisit untuk keamanan yang lebih baik

Membuat Security Profile WPA2 melalui Winbox

Dari sumber Go Wireless NZ (2023), cara paling mudah untuk membuat security profile WPA2 pada MikroTik adalah menggunakan antarmuka grafis Winbox. Langkah-langkahnya adalah sebagai berikut:

  1. Buka Winbox dan hubungkan ke router MikroTik Anda
  2. Buka menu Wireless dari sidebar
  3. Pilih tab Security Profiles
  4. Klik tombol + (plus) untuk menambahkan profile baru
  5. Isi kolom Name dengan nama profile (misalnya "profile-wpa2")
  6. Pilih dynamic-keys pada Mode
  7. Centang WPA2 PSK pada Authentication Types
  8. Masukkan password pada kolom WPA2 Pre-Shared Key
  9. Klik OK untuk menyimpan

Dari sumber MikroTik Wiki (2024), untuk keamanan optimal, pastikan bahwa parameter tambahan seperti berikut diatur dengan benar pada security profile:

/interface wireless security-profiles
print detail
# Melihat detail lengkap security profile yang ada
Flags: * - default
0 * name="default" mode=dynamic-keys authentication-types=wpa-psk,wpa2-psk
unicast-ciphers=aes-ccm,tkip group-ciphers=aes-ccm,tkip
wpa-pre-shared-key="12345678" wpa2-pre-shared-key="12345678"
group-key-update=5m
1 name="profile-wpa2" mode=dynamic-keys authentication-types=wpa2-psk
unicast-ciphers=aes-ccm group-ciphers=aes-ccm
wpa2-pre-shared-key="StrongPassw0rd!"
group-key-update=5m management-protection=allowed

Pastikan untuk menggunakan password yang kuat untuk WPA2 Pre-Shared Key. Password sebaiknya memiliki minimal 12 karakter dengan kombinasi huruf besar, huruf kecil, angka, dan karakter khusus untuk mencegah serangan brute force.

Dari 2 pendapat tersebut dapat disimpulkan bahwa pembuatan security profile WPA2 pada MikroTik dapat dilakukan melalui CLI maupun antarmuka grafis Winbox, dengan memperhatikan parameter-parameter penting seperti mode, authentication-types, dan cipher yang digunakan. Penggunaan mode dynamic-keys dengan authentication-types wpa2-psk dan cipher aes-ccm (AES) memberikan tingkat keamanan yang optimal untuk jaringan wireless.

3. MENERAPKAN SECURITY PROFILE KE INTERFACE WIRELESS

Tampilan topologi di gns3

Konfigurasi Interface Wireless melalui CLI

Dari sumber MikroTik Documentation (2024), setelah membuat security profile, langkah selanjutnya adalah menerapkannya ke interface wireless. Berikut adalah perintah CLI untuk mengkonfigurasi dan menerapkan security profile WPA2 ke interface wireless:

/interface wireless
set wlan1 mode=ap-bridge ssid="MikroTik-Secure" band=2ghz-b/g/n
channel-width=20/40mhz-Ce frequency=auto security-profile="profile-wpa2"
enable wlan1
# Mengkonfigurasi wlan1 sebagai access point dengan security profile "profile-wpa2"

Dari sumber RouterOS First Time Configuration (2023), untuk memastikan pengaturan interface wireless telah diterapkan dengan benar, Anda dapat menggunakan perintah berikut untuk memeriksa konfigurasi:

/interface wireless print
# Menampilkan konfigurasi semua interface wireless
Flags: X - disabled, R - running
0 R name="wlan1" mtu=1500 mac-address=D4:CA:6D:XX:XX:XX arp=enabled
interface-type=Atheros AR9300 mode=ap-bridge ssid="MikroTik-Secure"
frequency=2412 band=2ghz-b/g/n channel-width=20/40mhz-Ce
security-profile="profile-wpa2" radio-name="MikroTik"
wps-mode=disabled

Menerapkan Security Profile melalui Winbox

Dari sumber ShellHacks (2022), untuk menerapkan security profile ke interface wireless menggunakan Winbox, ikuti langkah-langkah berikut:

  1. Buka Winbox dan hubungkan ke router MikroTik
  2. Buka menu Wireless dari sidebar
  3. Pilih tab Interfaces
  4. Double-klik pada interface wireless yang ingin dikonfigurasi (misalnya wlan1)
  5. Pada tab Wireless, atur parameter berikut:
    • Mode: ap bridge
    • SSID: MikroTik-Secure (atau nama jaringan yang diinginkan)
    • Band: 2GHz-B/G/N (sesuaikan dengan hardware)
    • Channel Width: 20/40MHz Ce
  6. Pada dropdown Security Profile, pilih profile WPA2 yang telah dibuat sebelumnya
  7. Klik OK untuk menyimpan perubahan
  8. Kembali ke daftar interface, pastikan interface wireless telah diaktifkan (tidak ada tanda X)

Dari sumber NetworkWorld (2023), untuk memverifikasi bahwa konfigurasi wireless dan security profile telah diterapkan dengan benar, Anda dapat menggunakan menu Registration di Winbox untuk melihat klien yang terhubung:

/interface wireless registration-table print
# Menampilkan daftar klien yang terhubung ke wireless interface
# INTERFACE RADIO-NAME MAC-ADDRESS AP SIGNAL-STRENGTH TX-RATE
0 wlan1 client1 F4:B7:E2:XX:XX:XX no -62dBm 65.0Mbps

Pastikan untuk memantau registration-table secara berkala untuk mendeteksi adanya koneksi yang tidak dikenal. Pengguna yang tidak sah mungkin dapat terhubung jika password terlalu lemah atau jika keamanan jaringan telah dikompromikan.

Dari 2 pendapat tersebut dapat disimpulkan bahwa penerapan security profile WPA2 ke interface wireless merupakan langkah penting dalam mengamankan jaringan nirkabel. Konfigurasi dapat dilakukan melalui CLI atau Winbox dengan memperhatikan parameter seperti mode, band, channel width, dan SSID. Pemantauan registration-table membantu administrator dalam memverifikasi bahwa hanya perangkat yang sah yang terhubung ke jaringan.

4. PENGUJIAN DAN PRAKTIK KEAMANAN TERBAIK

Tampilan topologi di gns3

Pengujian Konfigurasi WPA2

Dari sumber TechTarget (2024), pengujian konfigurasi WPA2 adalah langkah penting untuk memastikan bahwa keamanan jaringan wireless berfungsi dengan baik. Beberapa metode pengujian meliputi:

# 1. Cek status konfigurasi security profile
/interface wireless security-profiles print
# 2. Verifikasi penerapan security profile ke interface
/interface wireless print
# 3. Coba koneksi dari perangkat klien dan verifikasi enkripsi
/interface wireless registration-table print
# 4. Memeriksa log untuk peristiwa autentikasi
/log print where topics~"wireless"

Dari sumber MikroTik Forum (2022), pengujian keamanan WPA2 sebaiknya juga mencakup pemeriksaan terhadap potensi kelemahan. Beberapa pengujian tambahan meliputi:

# 5. Verifikasi kekuatan sinyal dan interferensi channel
/interface wireless scan
# 6. Periksa aktivitas mencurigakan pada registration-table
/interface wireless registration-table print stats
# 7. Cek traffic interface wireless untuk deteksi anomali
/interface monitor-traffic wlan1 once

Praktik Keamanan Terbaik untuk WPA2

Dari sumber Meraki Documentation (2020), berikut adalah beberapa praktik keamanan terbaik untuk konfigurasi WPA2 pada jaringan wireless MikroTik:

  1. Gunakan password yang kuat dan panjang (minimal 12 karakter) dengan kombinasi huruf, angka, dan simbol
  2. Aktifkan management-protection untuk mencegah serangan deauthentication
  3. Gunakan unicast-ciphers dan group-ciphers aes-ccm saja, hindari penggunaan TKIP
  4. Perbarui firmware MikroTik secara teratur untuk mendapatkan perbaikan keamanan terbaru
  5. Atur group-key-update ke nilai yang tepat (default 5 menit sudah baik)
# Contoh konfigurasi security profile dengan praktik terbaik
/interface wireless security-profiles
add name="best-practice" mode=dynamic-keys authentication-types=wpa2-psk
unicast-ciphers=aes-ccm group-ciphers=aes-ccm
wpa2-pre-shared-key="V3ryStr0ngP@ssw0rd!" group-key-update=5m
management-protection=required wps-mode=disabled

Dari sumber Strong.eu Blog (2023), pada jaringan modern, pertimbangkan untuk menggunakan mode transisi WPA2/WPA3 jika perangkat mendukung, untuk memperoleh keamanan yang lebih baik sambil mempertahankan kompatibilitas dengan perangkat lama:

# Konfigurasi untuk mode transisi WPA2/WPA3 pada RouterOS v7+
/interface wireless security-profiles
add name="transition-mode" mode=dynamic-keys
authentication-types=wpa2-psk,wpa3-psk
unicast-ciphers=aes-ccm group-ciphers=aes-ccm
wpa2-pre-shared-key="V3ryStr0ngP@ssw0rd!"
wpa3-pre-shared-key="V3ryStr0ngP@ssw0rd!"
management-protection=required wps-mode=disabled

Untuk RouterOS versi 7 ke atas, konfigurasi wireless pada wifiwave2 package menggunakan sintaks yang sedikit berbeda. Metode autentikasi dan enkripsi dikonfigurasi menggunakan path /interface/wifi/security/ dengan parameter seperti authentication-types, encryption, dan passphrase.

Meskipun WPA2 masih cukup aman jika dikonfigurasi dengan benar, namun WPA2 memiliki kelemahan seperti kerentanan KRACK (Key Reinstallation Attack). Pertimbangkan untuk beralih ke WPA3 jika perangkat klien Anda mendukungnya, terutama untuk jaringan yang menangani data sensitif.

Dari 2 pendapat tersebut dapat disimpulkan bahwa pengujian dan penerapan praktik keamanan terbaik merupakan langkah penting dalam mengamankan jaringan wireless dengan WPA2. Pengujian harus mencakup verifikasi konfigurasi, pemeriksaan koneksi klien, dan pemantauan aktivitas jaringan. Praktik terbaik meliputi penggunaan password yang kuat, enkripsi AES, perlindungan manajemen, dan pembaruan firmware secara teratur, serta pertimbangan untuk beralih ke mode transisi WPA2/WPA3 jika perangkat mendukung.
